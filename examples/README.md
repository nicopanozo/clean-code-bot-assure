# Before / after examples

These pairs are **mock implementations** written to look like real production deltas: the `*_before`
files resemble rushed legacy modules; the matching `*_after` files show the kind of structure,
typing, and documentation the Clean Code Bot is meant to produce. They are **not** generated by
the model—they are fixed fixtures for coursework, demos, and regression spot-checks.

| Before (input-style) | After (target-style) | Notes |
| --- | --- | --- |
| `order_service_before.py` | `order_service_after.py` | Monolith checkout: opaque dict errors, dense `go()` method. |
| `rent_roll_import_before.py` | `rent_roll_import_after.py` | Property onboarding CSV: bare `except`, manual indexing, implicit contracts. |
| `stripe_webhook_handler_before.js` | `stripe_webhook_handler_after.js` | Billing webhook: ad-hoc signature parsing, inline `require`, loose equality. |

## Try the CLI on a “before” file

```bash
clean-code-bot refactor examples/order_service_before.py -o /tmp/order_service_refactored.py
clean-code-bot refactor examples/rent_roll_import_before.py -o /tmp/rent_roll_refactored.py
clean-code-bot refactor examples/stripe_webhook_handler_before.js -o /tmp/stripe_webhook_refactored.js
```

Compare the tool output with the checked-in `*_after` files to judge how close your configured model
gets to the intended style.
